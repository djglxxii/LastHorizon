# T006 Armada Checklist

- [x] Spawner emits successive blocks on a tunable cadence.
- [x] Two blocks coexist during normal play at the default tuning; see the live scene capture `armada-descent-02-two-blocks.png`.
- [x] Each block descends as a rigid unit; the block has no horizontal step, scatter, dive, or rusher behavior.
- [x] Individual enemies sway horizontally with independent phase offsets. `sway-detail.png` is a zoomed crop from the live scene and shows neighboring enemies out of lockstep.
- [x] Enemies despawn silently after the formation crosses below the bottom of the playfield.
- [x] Pea shooter and typed weapon continue firing across the armada with no damage interaction.
- [x] Player collision response is intentionally absent in T006.
- [x] Baseline enemy uses `assets/sprites/enemies/baseline-grunt.png`, generated by `tools/generate_baseline_enemy_sprite.gd`.
- [x] Entity, formation, and spawner scripts live under `src/enemies/`.
- [x] Block dimensions, descent speed, sway amplitude/period, and spawn cadence are exported tuning fields.
- [x] `max_hp` is exported on `BaselineEnemy`; no damage code is wired.
- [x] No enemy bullets, rushers, elite tier, Grid meter, carriers, or faction variants were introduced.
- [x] Headless smoke passes and prints `LAST_HORIZON_BOOT_SMOKE_OK`.
- [x] Visual evidence was captured from the running `Main.tscn` scene tree by `capture_live_armada_evidence.gd`, not by reimplementing formation movement in a separate renderer.
